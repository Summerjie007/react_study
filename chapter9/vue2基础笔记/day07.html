HoreEdit.vue 的完整代码

<template>
  <div class="hore-edit">
    <h3>编辑英雄</h3>
    <hr />
    <form class="form-horizontal" @submit.prevent="editHore()">
      <div class="form-group">
        <label class="col-sm-2 control-label">英雄名称</label>
        <div class="col-sm-10">
          <input
            v-model="horeForm.horeName"
            type="text"
            class="form-control"
            style="width:200px"
          />
        </div>
      </div>
      <div class="form-group">
        <label class="col-sm-2 control-label">英雄性别</label>
        <div class="col-sm-10">
          <input v-model="horeForm.gender" type="radio" value="男" /> 男
          <input v-model="horeForm.gender" type="radio" value="女" /> 女
        </div>
      </div>
      <div class="form-group">
        <div class="col-sm-offset-2 col-sm-10">
          <button type="submit" class="btn btn-primary">编辑</button>
        </div>
      </div>
    </form>
  </div>
</template>

<script>
  import axios from "axios";
  export default {
    data() {
      return {
        horeForm: {
          horeName: "",
          gender: ""
        }
      };
    },
    created() {
      this.getHore();
    },
    methods: {
      // 根据ID或英雄详情，填充表单
      getHore() {
        // this.$route.params.id 是地址栏上的ID
        axios
          .get("http://localhost:3000/hore/" + this.$route.params.id)
          .then((res) => {
            // 填充表单
            this.horeForm = res.data;
          });
      },
      // 编辑英雄
      editHore() {
        // horeName  gender time 需要三个数据，在请求体中
        // this.horeForm  拥有四个数据
        const { horeName, gender, id } = this.horeForm;
        axios
          .put("http://localhost:3000/hore/" + id, {
            horeName,
            gender,
            time: new Date()
          })
          .then(() => {
            // 修改成功
            this.$router.push("/hore");
          });
      }
    }
  };
</script>

# 04-案例优化-axios 全局使用

痛点：

  • 对应 axios 而言，基本上每个组件都要使用它。
  • 那么每个组件都需要，先导入，后使用。

方案：

  • 如果每一个 vue 实例，都能直接访问到 axios 那该多好啊。
  • 大白话：在每一个 vue 的实例下拥有一个属性，这个属性指向 axios 对象即可。
  • 可以在 Vue 这个构造函数的原型上，加一个属性，将来所有 vue 实例均可访问这个属性。

代码：

main.js 给 Vue 添加原型属性指向 axios

import axios from "axios";
// 给Vue原型添加属性，将来的vue实例可访问
Vue.prototype.$http = axios;

其他组件，注册导入 axios 的代码，将 axios 改成 this.$http 即可。

# 05-案例优化-axios 基准地址

痛点：

  • 接口服务器的地址可能会变动，接口地址后面的 resetful 规则一般不会变。
  • 地址的前面一段：协议+域名+端口号，修改改所有的组件都需要修改，很麻烦。

基准地址：

  • 接口地址前面一段相同的路径，基准地址。

想法：

  • 把相同的地址声明成变量，将来进行动态拼接即可。

方案：

  • axios 提供了基准地址的配置，一旦配置好，将来发请求的时候会自动拼接基准地址。
  • 发请求的时候，只需要书写简短的路径即可。

代码：

  • baseURL 用来配置基准路径
    • ‘/hore’ 在前面自动加上 baseURL 配置的路径
    • ‘http://localhost:3000/hore’ 不会加
  • 参考：axios.defaults.baseURL = 'https://api.example.com';

main.js

// 给axios配置默认的基准路径
axios.defaults.baseURL = "http://localhost:3000";

# 07-vue 扩展-动画

vue 实现的动画不会特别复杂，如果要实现炫酷的动画：直接 css3 canvas 技术实现。

例如：元素，组件的移除创建，元素，组件的显示和隐藏，可以使用 vue 的动画。

理解：在 vue 的进入和离开的时候有 6 个状态，会给元素加上对应的 class 类名。

  • Enter 是元素的创建，显示。
    • 进入前 默认 v-enter
    • 进入中 默认 v-enter-active
    • 进入后 默认 v-enter-to
  • Leave 是元素的移除，隐藏。
    • 离开前 默认 v-leave
    • 离开中 默认 v-leave-active
    • 离开后 默认 v-leave-to

在不同的类中，通过加过渡且去修改 css 属性，去触发过渡从而实现动画。

transition 是 css3 的过渡

  • 先加上 transition 属性 ，例如： transition:all 1s 任何属性的改变触发 1s 中的动画来完成。
  • 当属性发生改变的时候，就会触发过渡效果，也就是动画效果。

单个元素动画

在 vue 中，给需要做动画的元素，加上 transition 标签（组件），我们的元素在进入和离开的时候，会拥有 6 个状态自动的切换类名，我们要做的就是在这个 6 个类名通过属性变化和过渡属性的添加来完成动画。

        width: 100px;

        background: green;
        transition: all 1s;
      }

        width: 200px;

      /* 6个状态对应6个类 */

        transition: all 2s;

列表动画控制

列表的意思是同 v-for 渲染出来的元素称为列表。为这些元素实现进入和离开动画。

<!DOCTYPE html>
<html>
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title></title>
    <style>
      ul {
        list-style: none;
        width: 400px;
        padding: 0;
        margin: 100px auto;
      }
      li {
        width: 400px;
        height: 40px;
        line-height: 40px;
        border: 1px dashed #ccc;
        padding: 0 10px;
        background: #eee;
      }
      /* 进入：淡入,底部100px处上移 */
      /* 离开：淡出,下移底部100px处 */
      .v-enter,
      .v-leave-to {
        opacity: 0;
        transform: translateY(100px);
      }
      .v-enter-active,
      .v-leave-active {
        transition: all 1s;
        /* 让动画的元素不占位 */
        position: absolute;
      }
      /* 列表才有的状态类名 */
      /* 列表元素移动的时候，加动画 */
      .v-move {
        transition: all 1s;
      }
    </style>
  </head>
  <body>
    <div id="app">
      <transition-group tag="ul">
        <li v-for="item in list" :key="item.id" @click="del(item.id)">
          {{item.name}}
        </li>
      </transition-group>
    </div>
    <script src="./vue.js"></script>
    <script>
      const vm = new Vue({
        el: "#app",
        data: {
          list: [
            { id: 1, name: "周杰伦" },
            { id: 2, name: "高圆圆" },
            { id: 3, name: "刘德华" },
            { id: 4, name: "张学友" }
          ]
        },
        methods: {
          del(id) {
            const index = this.list.findIndex((item) => item.id === id);
            this.list.splice(index, 1);
          }
        }
      });
    </script>
  </body>
</html>

注意：

  • tag 可以指定 transition-group 解析成什么元素
  • v-move 列表元素移动的时候一个状态类，可以加动画
  • 动画元素不能占位置，这样才能动画移上去

# 08-vue 扩展-插槽

插槽：在封装组件的时候，可以让组件的复用度更高。

画图理解：

解释一下：

  • 当你在定义组件的时候，可以留下一处插槽
  • 当前在使用组件的时候，插入不同的内容，提高组件复用度。

代码演示：

  • 默认插槽 (当组件只有一处内容需要插入的时候)
<div id="app">
  <!-- 组件标签中的内容，就是去填入插槽的内容。 -->
  <com-page>页面1</com-page>
  <com-page>页面2</com-page>
  <com-page>页面3</com-page>
</div>
<script src="./vue.js"></script>
<script>
  // 定义页面组件 com-page
  Vue.component("com-page", {
    template: `<div class="com-page">
          <header>头部</header>
          <section>
            <!-- 插槽（默认插槽） -->
            <slot></slot>
          </section>  
          <footer>底部</footer>
        </div>`
  });

  const vm = new Vue({
    el: "#app"
  });
</script>
  • 后备内容 (当你使用组件的时候，并没有插入内容，但是需要默认内容)
<slot>后备内容：页面</slot>
<!-- slot标签中的内容，就是后备内容。没有给插槽插入内容的内容会显示 -->
  • 具名插槽 (当组件有多处内容需要插入的时候)
<div id="app">
  <!-- 组件标签中的内容，就是去填入插槽(默认)的内容。 -->
  <com-page>
    <!-- 具名插槽使用：slot属性来指定插槽 -->
    <div slot="section">页面xxx</div>
    <div slot="footer">底部xxx</div>
  </com-page>
  <!-- <com-page>页面2</com-page>
      <com-page></com-page> -->
</div>

# 09-vue 扩展-混入

混入：当每个组件拥有一些相同的业务逻辑，没给组件就会写相同的代码，代码冗余。你可以把相同的业务逻辑抽离出来，混入（放到）到不同的组件中去使用。混入：mixin

less 的混入

vue 的混入

  • 全局
<!DOCTYPE html>
<html>
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title></title>
  </head>
  <body>
    <div id="app">
      <com-a></com-a>
      <com-b></com-b>

    <script>
      // 全局：定义混入的逻辑，将来其他组件都会拥有
      Vue.mixin({
        // vue 配置选项

            alert("你好");

        }
